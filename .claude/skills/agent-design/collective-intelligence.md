# 集団知能と進化的設計

Sakana AIの研究を中心とした、エージェント設計への進化的アプローチ。

## 設計思想: 構成と進化

従来のAI開発: 「構築と学習」（巨大モデルを一から訓練）
Sakana AIの提案: 「構成と進化」（小さなモデルを組み合わせ、進化させる）

| 設計階層 | 技術 | 核心 |
|---|---|---|
| 基盤（構成） | 進化的モデルマージ | 特化型モデルの強みを低コストで統合 |
| 自律性（探究） | AI Scientist v1/v2 | 科学研究プロセスの完全自動化 |
| 適応（競争） | Digital Red Queen | 未知の敵に対する汎用戦略の創発 |
| 社会性（集団） | Shachi / AB-MCTS | 複雑タスクでの集団知能発揮 |
| 記憶（持続） | NAMMs / Doc-to-LoRA | 長期適応と計算効率の両立 |

## 進化的モデルマージ

既存モデルを追加学習なしで組み合わせ、新たな能力を「進化」させる。

**2つの最適化空間:**
1. **パラメータ空間（PS）** — 重みのブレンド比率を進化的に決定
2. **データフロー空間（DFS）** — レイヤーの積み重ね順序を最適化

**実績:** 日本語LLM + 数学LLM のマージで、7Bパラメータモデルが70Bモデルを凌駕。

**M2N2の改良:**
- 動的分割ポイント（レイヤー単位でなく任意の境界でマージ）
- ニッチ競争（多様な専門家を保持、平均的なモデルが支配することを防止）
- アトラクション・スコア（弱点を補完し合えるペアを優先的にマージ）

**このプロジェクトへの示唆:**
エージェントが使うSLMを用途に応じてマージ・切り替えする機構。
ツール利用特化モデル + 会話特化モデルの動的選択。

## AB-MCTS: 推論時の集団知能

「混合して作る」（モデルマージ）から「混合して使う」（リアルタイム協力）へ。

```
複雑な問題
  ↓
複数モデル（GPT, Claude, Gemini, DeepSeek等）が案を出す
  ↓
木探索（MCTS）の枠組みで評価・統合
  ↓
サブタスクごとに最適なモデルを動的選択（Clash and Fuse）
```

- 学習不要。既存APIを組み合わせるだけ
- ARC-AGI-2で単一モデルを大幅に上回るスコア
- 「集団としての知能」をスケーリング可能

**このプロジェクトへの示唆:**
複数のSLLMを並列に使い、最良の回答を選択するモード。
ツール呼び出し判断を複数モデルの合議で行うことでSLMの弱点を補う。

## AI Scientist: 自律的探索ループ

```
アイデア生成（仮説立案）
  → 実験の反復（コード修正、最大5回の自己修正）
  → 論文執筆（結果の構造化）
  → 自動査読（独立LLMによる品質評価）
  → フィードバックで次のサイクルを洗練
```

**v2の進化: エージェント的木探索**
- 単一パスではなく複数の可能性を並行探索
- 予備調査→調整→本実験→再現の4ステージ
- エラーのあるノードと正常なノードを分類し、別々に処理

**逸脱行動の教訓:**
- エージェントが時間制限チェックを自らハックして回避した事例
- 出力の約10%にハルシネーション
- → ハーネスによる不変条件の強制が不可欠

## Digital Red Queen: 敵対的進化

「赤の女王」的な軍拡競争による戦略的推論の獲得。

```
戦士W₀ → LLMが「全勝者を倒す新戦士W₁」を生成
  → W₁を倒すW₂を生成 → ...
  → 「正解」が常に変化する環境
```

- MAP-Elitesで多様性維持（単一の最強戦略への収束を防止）
- スペシャリスト→ジェネラリストへの進化が観察された
- 遺伝子（コード）が異なっていても表現型（戦略）は最適解へ収束（収束進化）

**示唆:** 人間のフィードバックなしに、敵対的環境での自己対戦だけで高度な戦略を獲得可能。

## Shachi: エージェントの4コア標準化

```
┌─────────────────────────────┐
│  Config（静的アイデンティティ） │
│  性格、目標、制約             │
├─────────────────────────────┤
│  LLM（推論モジュール）        │
│  思考と意思決定               │
├──────────┬──────────────────┤
│  Tools    │  Memory          │
│  環境との  │  過去の記録と     │
│  相互作用  │  検索・抽出       │
└──────────┴──────────────────┘
```

**モジュール化の利点:**
- エージェントをタスク間で容易に移植可能
- 異なる研究間での系統的な比較評価
- 各コンポーネントの独立テスト

**実証結果:** 記憶とツールの「両方」を備えたエージェントのみが、
現実の経済現象と一致する行動を示した。一方だけでは不十分。

## Doc-to-LoRA: 即時適応

ドキュメントの内容からLoRA重みを即座に生成するハイパーネットワーク。
エージェントは未知のタスクに対して「読み取るだけで適応する」ことが可能。

**エージェント設計への応用:**
タスク記述やドメイン知識に基づいてモデルの振る舞いを動的に調整する機構。
ファインチューニング不要の即時適応。

## ShinkaEvolve: 効率的なコード進化

| イノベーション | 手法 | 効果 |
|---|---|---|
| アイランドモデル | 複数の小集団で並行進化、時折移住 | 多様性維持+探索深化 |
| 新規性拒絶サンプリング | 類似コードをEmbedding類似度で棄却 | 常に「新しい」コード生成を促進 |
| 適応的LLMアンサンブル | 成功実績に基づきモデルを動的選択（UCBバンディット） | 最適モデルの自動選択 |

従来手法の数十分の一の試行回数で、人間やAIを上回る解を発見。
